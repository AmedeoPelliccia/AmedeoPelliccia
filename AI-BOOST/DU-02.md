# DU-02 — Adversarial Misuse of Open-Weights Model

> **Programme:** AI-BOOST — GA 101135737 (EuroHPC JU)
> **Parent deliverable:** [DEL-07 Ethics and Security](DEL-07-ethics-and-security.md)
> **Machine-readable source:** [`ethics-and-security.yaml`](ethics-and-security.yaml) § `dual_use.risk_analysis[1]`
> **Related:** [Dual-Use Risk Analysis](dual-use-risk-analysis.md)

---

## 1. Risk Description

Open-weights release could enable generation of harmful content.

## 2. Risk Assessment

| Dimension | Rating |
|-----------|--------|
| Likelihood | **Medium** |
| Impact | Medium |
| Residual risk (post-mitigation) | Low–Medium |

## 3. Mitigation

- Red-teaming prior to release (WP4).
- Safety fine-tuning applied before open-weights distribution.
- Responsible release protocol with staged rollout.
- Input sanitisation layer and output filtering for safety-critical deployments.

## 4. Export Control

| Requirement | Status | Evidence |
|-------------|:------:|----------|
| EU Dual-Use Regulation (EU) 2021/821 screening | Planned | All consortium partners screened; no listed entities |
| US EAR/ITAR assessment | Planned | No US-origin controlled technology used in core pipeline |
| End-use monitoring | Planned | Responsible use license terms for model distribution |

## 5. Verification

- Red-team evaluation report produced before each release.
- Safety filter specification documented.
- Residual risk downgraded after responsible release protocol is validated.

## 6. Review Schedule

Re-assessed at each consortium review (M6, M18, M36). Will be downgraded after completion of WP4 red-teaming.
