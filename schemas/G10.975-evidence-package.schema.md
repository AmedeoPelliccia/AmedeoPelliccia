## 1. Purpose

This schema defines the **canonical signed-YAML form** of every Evidence Package produced under G10.975 §8. An Evidence Package is the authoritative record by which a `G10.97x` entity's containment state is established, escalated, exited from quarantine, or activated.

Every transition that requires evidence under the BREX rules
([`GQAOA-UTA-G10-975-BREX-RULES-001`](../OPT-INS_FRAMEWORK/GQAOA-UTA-G10-975-BREX-RULES-001.yaml))
must be backed by a document conforming to this schema.

## 2. Top-level structure

The document has a single root key `evidence_package` whose required members are:

| Field | Type | Notes |
|-------|------|-------|
| `id` | string | Pattern `G10.975-EVIDENCE-YYYYMMDD-NNN` |
| `entity` | object | `{id, name}` — `id` ∈ `G10.970`–`G10.979` |
| `containment_state` | enum | One of the six §3 states |
| `source_hash` | string | `sha256` (64 hex) or literal placeholder `<sha256>` |
| `quarantine_trigger` | string *(optional)* | Required when entering or exiting QUARANTINED |
| `residual_target` | string | LC01-style residual-risk target |
| `authority` | object | STK approvals (see §4) |
| `decision` | object | `{value, reviewer, timestamp}` |
| `ledger` | object | `{anchor_id, csv_row_ref?}` — anchors evidence into the audit ledger |

`additionalProperties` is `false` at every level — unknown keys fail validation.

## 3. Containment state enum

The schema enumerates exactly the six states defined in the normative spec §3:

`OBSERVE_ONLY`, `SANDBOXED`, `QUARANTINED`, `REGENCY_REVIEW`, `CONTAINED_ACTIVE`, `RETIRED`.

The `decision.value` enum mirrors the same six states so the recorded decision can only target a defined containment lattice node.

## 4. Authority block

| Key | Constant value | Required |
|-----|----------------|----------|
| `gov` | `STK-GOV` | yes |
| `safety` | `STK-SAFETY` | yes |
| `ledger` | `STK-LEDGER` | yes |
| `qcsaa` | `STK-QCSAA` | optional |
| `ethics` | `STK-ETHICS` | optional |

`STK-GOV` is the named **enforcement authority** under §11 of the normative spec. `STK-SAFETY` and `STK-LEDGER` are non-negotiable co-signers for any package supporting `CONTAINED_ACTIVE`.

## 5. Identifiers and patterns

- `evidence_package.id` — `^G10\.975-EVIDENCE-[0-9]{8}-[0-9]{3,}$`
- `entity.id` — `^G10\.97[0-9]$`
- `source_hash` — `^(<sha256>|[A-Fa-f0-9]{64})$`

The placeholder `<sha256>` is permitted only in draft templates; production records must contain a real digest.

## 6. Validation

Run:

```bash
python tools/validators/validate_g10_975.py
```

The validator loads every Evidence Package referenced from the BREX/registry and rejects packages that:

- omit any required field,
- introduce additional properties,
- declare a state outside the enum,
- attempt a transition disallowed by the permitted-transitions matrix,
- declare `CONTAINED_ACTIVE` without a `ledger.anchor_id`.
